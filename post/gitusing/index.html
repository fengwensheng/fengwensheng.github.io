Git使用

2020-07-05

记录一下常用的Git命令和操作流。

1.最常用

git clone targetUrl

clone远程项目到本地。clone时会一并将项目最外层的文件夹一并拉下来，不用担心会散落一地。

2.当前所在分支

git branch

3.历史提交记录

git log
git log –oneline

使用 git log --oneline 查看起来更友好一些，去掉了Author和Date。Commit ID也只是显示前7位，使得一行放一条记录。

4.本地项目如何回到历史提交版本

git reset –hard commitId

这时就可以配合查看远程服务器上的历史提交记录的命令一起使用了。

5.初始化本地仓库

git init

6.本地代码加入本地仓库

git add .

7.把本地修改提交本地版本库及备注

git commit -m “commitString”

8.本地库与远程平台关联

git remote add origin targetUrl

这个操作只需第一次push前操作一次即可。

9.推送本地提交到远程平台命令

git push origin master

提交到master分支。

10.第一次push时不成功，提示要pull等

不管它，直接用 -f 来push。

11.后续每次更新远程的操作流

git add .
git commit -m “commitString”
git push origin master

git add . 即完成了staged效果。
